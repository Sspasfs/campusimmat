\doxysubsection{Description détaillée}
\doxylink{class_plugin}{Plugin} manager implementation for translation loaders.

Enforces that loaders retrieved are either instances of Loader\textbackslash{}\+File\+Loader\+Interface or Loader\textbackslash{}\+Remote\+Loader\+Interface. Additionally, it registers a number of default loaders.

If you are wanting to use the ability to load translation files from the include\+\_\+path, you will need to create a factory to override the defaults defined in this class. A simple factory might look like\+:

{\ttfamily  function (\$translators) \{ \$adapter = new Gettext(); \$adapter-\/\texorpdfstring{$>$}{>}set\+Use\+Include\+Path(true); return \$adapter; \} }

You may need to override the Translator service factory to make this happen more easily. That can be done by extending it\+:

{\ttfamily  use Laminas\textbackslash{}\+I18n\textbackslash{}\+Translator\textbackslash{}\+Translator\+Service\+Factory; // or Laminas\textbackslash{}\+Mvc\textbackslash{}\+I18n\textbackslash{}\+Translator\+Service\+Factory use Laminas\textbackslash{}\+Service\+Manager\textbackslash{}\+Service\+Locator\+Interface;}

{\ttfamily class My\+Translator\+Service\+Factory extends Translator\+Service\+Factory \{ public function create\+Service(Service\+Locator\+Interface \$services) \{ \$translator = parent\+::create\+Service(\$services); \$translator-\/\texorpdfstring{$>$}{>}\doxylink{report_8contract_8php_a77b973d137fb33212e018b042df6e3e7}{get\+Loader\+Plugin\+Manager()}-\/\texorpdfstring{$>$}{>}set\+Factory(...); return \$translator; \} \} }

You would then specify your custom factory in your service configuration.
