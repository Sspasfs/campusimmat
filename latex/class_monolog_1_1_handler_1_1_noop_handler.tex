\doxysubsection{Description détaillée}
No-\/op

This handler handles anything, but does nothing, and does not stop bubbling to the rest of the stack. This can be used for testing, or to disable a handler when overriding a configuration without influencing the rest of the stack.

\doxysubsubsection{\texorpdfstring{handle()}{handle()}}
{\footnotesize\ttfamily handle (\begin{DoxyParamCaption}\item[{array}]{\$record }\end{DoxyParamCaption})}

Handles a record.

All records may be passed to this method, and the handler should discard those that it does not want to handle.

The return value of this function controls the bubbling process of the handler stack. Unless the bubbling is interrupted (by returning true), the \doxylink{class_monolog_1_1_logger}{Logger} class will keep on calling further handlers in the stack with a given log record.


\begin{DoxyParams}[1]{Paramètres}
array & {\em \$record} & The record to handle \\
\hline
\end{DoxyParams}
\begin{DoxyReturn}{Renvoie}
bool true means that this handler handled the record, and that bubbling is not permitted. false means the record was either not processed or that this handler allows bubbling.

{\footnotesize\ttfamily is\+Handling (\begin{DoxyParamCaption}\item[{array}]{\$record }\end{DoxyParamCaption})}

Checks whether the given record will be handled by this handler.

This is mostly done for performance reasons, to avoid calling processors for nothing.

Handlers should still check the record levels within \doxylink{interface_monolog_1_1_handler_1_1_handler_interface_ad54b568c673b7b4fc724268a54313603}{handle()}, returning false in \doxylink{interface_monolog_1_1_handler_1_1_handler_interface_abbfbbcd3430dea8422ecd3c725737758}{is\+Handling()} is no guarantee that \doxylink{interface_monolog_1_1_handler_1_1_handler_interface_ad54b568c673b7b4fc724268a54313603}{handle()} will not be called, and \doxylink{interface_monolog_1_1_handler_1_1_handler_interface_abbfbbcd3430dea8422ecd3c725737758}{is\+Handling()} might not be called for a given record.
