\doxysubsection{Description détaillée}
Uri element.

This represents a single uri. An example of how this may be encoded\+:

$<$link$>$/foo/bar$<$/link$>$ \texorpdfstring{$<$}{<}d\+:href xmlns\+:d="{}\+DAV\+:"{}\texorpdfstring{$>$}{>}\href{http://example.org/hi}{\texttt{ http\+://example.\+org/hi}}\texorpdfstring{$<$}{<}/d\+:href\texorpdfstring{$>$}{>}

If the uri is relative, it will be automatically expanded to an absolute url during writing and reading, if the context\+Uri property is set on the reader and/or writer.

This method is called during xml parsing.

This method is called statically, this is because in theory this method may be used as a type of constructor, or factory method.

Often you want to return an instance of the current class, but you are free to return other data as well.

Important note 2\+: You are responsible for advancing the reader to the next element. Not doing anything will result in a never-\/ending loop.

If you just want to skip parsing for this element altogether, you can just call \$reader-\/\texorpdfstring{$>$}{>}next();

\$reader-\/\texorpdfstring{$>$}{>}\doxylink{report_8contract_8php_a77b973d137fb33212e018b042df6e3e7}{parse\+Sub\+Tree()} will parse the entire sub-\/tree, and advance to the next element.

The xml\+Serialize method is called during xml writing.

Use the \$writer argument to write its own xml serialization.

An important note\+: do {\itshape not} create a parent element. Any element implementing Xml\+Serializable should only ever write what\textquotesingle{}s considered its \textquotesingle{}inner xml\textquotesingle{}.

The parent of the current element is responsible for writing a containing element.

This allows serializers to be re-\/used for different element names.

If you are opening new elements, you must also close them again.
