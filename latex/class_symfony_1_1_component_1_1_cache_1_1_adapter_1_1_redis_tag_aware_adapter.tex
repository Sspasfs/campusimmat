\doxysubsection{Description détaillée}
Stores tag id \texorpdfstring{$<$}{<}\texorpdfstring{$>$}{>} cache id relationship as a Redis Set.

Set (tag relation info) is stored without expiry (non-\/volatile), while cache always gets an expiry (volatile) even if not set by caller. Thus if you configure redis with the right eviction policy you can be safe this tag \texorpdfstring{$<$}{<}\texorpdfstring{$>$}{>} cache relationship survives eviction (cache cleanup when Redis runs out of memory).

Redis server 2.\+8+ with any {\ttfamily volatile-\/\texorpdfstring{$\ast$}{*}} eviction policy, OR {\ttfamily noeviction} if you\textquotesingle{}re sure memory will NEVER fill up

Design limitations\+:
\begin{DoxyItemize}
\item Max 4 billion cache keys per cache tag as limited by Redis Set datatype. E.\+g. If you use a "{}all"{} items tag for expiry instead of clear(), that limits you to 4 billion cache items also.
\end{DoxyItemize}
