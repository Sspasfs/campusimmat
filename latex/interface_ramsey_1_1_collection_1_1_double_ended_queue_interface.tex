\doxysubsection{Description détaillée}
A linear collection that supports element insertion and removal at both ends.

Most {\ttfamily Double\+Ended\+Queue\+Interface} implementations place no fixed limits on the number of elements they may contain, but this interface supports capacity-\/restricted double-\/ended queues as well as those with no fixed size limit.

This interface defines methods to access the elements at both ends of the double-\/ended queue. Methods are provided to insert, remove, and examine the element. Each of these methods exists in two forms\+: one throws an exception if the operation fails, the other returns a special value (either {\ttfamily null} or {\ttfamily false}, depending on the operation). The latter form of the insert operation is designed specifically for use with capacity-\/restricted implementations; in most implementations, insert operations cannot fail.

This interface extends the {\ttfamily Queue\+Interface}. When a double-\/ended queue is used as a queue, FIFO (first-\/in-\/first-\/out) behavior results. Elements are added at the end of the double-\/ended queue and removed from the beginning. Note that the {\ttfamily peek()} method works equally well when a double-\/ended queue is used as a queue or a stack; in either case, elements are drawn from the beginning of the double-\/ended queue.

While {\ttfamily Double\+Ended\+Queue\+Interface} implementations are not strictly required to prohibit the insertion of {\ttfamily null} elements, they are strongly encouraged to do so. Users of any {\ttfamily Double\+Ended\+Queue\+Interface} implementations that do allow {\ttfamily null} elements are strongly encouraged {\itshape not} to take advantage of the ability to insert nulls. This is so because {\ttfamily null} is used as a special return value by various methods to indicated that the double-\/ended queue is empty.
