Endpoint \+: Resource available though the API. The endpoint is the URL where your API can be accessed by a client application

Method \+: HTTP verbs to indicate the desired action to be performed on the identified resource. See\+: \href{https://en.wikipedia.org/wiki/Hypertext_Transfer_Protocol\#Request_methods}{\texttt{ https\+://en.\+wikipedia.\+org/wiki/\+Hypertext\+\_\+\+Transfer\+\_\+\+Protocol\#\+Request\+\_\+methods}}

itemtype \+: A \doxylink{class_g_l_p_i}{GLPI} type, could be an asset, an ITIL or a configuration object, etc. This type must be a class who inherits Common\+DTBM \doxylink{class_g_l_p_i}{GLPI} class.

search\+Option \+: A column identifier (integer) of an itemtype (ex\+: 1 -\/\texorpdfstring{$>$}{>} id, 2 -\/\texorpdfstring{$>$}{>} name, ...). See \doxylink{report_8contract_8php_a77b973d137fb33212e018b042df6e3e7}{List search\+Options} endpoint.

JSON Payload \+: content of HTTP Request in JSON format (HTTP body)

Query string \+: URL parameters

\doxylink{class_user}{User} token \+: Used in login process instead of login/password couple. It represent the user with a string. You can find user token in the settings tabs of users.

\doxylink{class_session}{Session} token \+: A string describing a valid session in glpi. Except init\+Session endpoint who provide this token, all others require this string to be used.

App(lication) token \+: An optional way to filter the access to the API. On API call, it will try to find an API client matching your IP and the app token (if provided). You can define an API client with an app token in general configuration for each of your external applications to identify them (each API client have its own history).\hypertarget{md_public_2glpi_2apirest_autotoc_md360}{}\doxysection{\texorpdfstring{Important}{Important}}\label{md_public_2glpi_2apirest_autotoc_md360}

\begin{DoxyItemize}
\item You should always provide a Content-\/\+Type header in your HTTP calls. Currently, the API supports\+:
\begin{DoxyItemize}
\item application/json
\item multipart/form-\/data (for files upload, see \doxylink{report_8contract_8php_a77b973d137fb33212e018b042df6e3e7}{Add item(s)} endpoint.
\end{DoxyItemize}
\item GET requests must have an empty body. You must pass all parameters in URL. Failing to do so will trigger an HTTP 400 response.
\item By default, sessions used in this API are read-\/only. Only some methods have write access to session\+:
\begin{DoxyItemize}
\item \doxylink{report_8contract_8php_a77b973d137fb33212e018b042df6e3e7}{init\+Session}
\item \doxylink{report_8contract_8php_a77b973d137fb33212e018b042df6e3e7}{kill\+Session}
\item \doxylink{report_8contract_8php_a77b973d137fb33212e018b042df6e3e7}{change\+Active\+Entities}
\item \doxylink{report_8contract_8php_a77b973d137fb33212e018b042df6e3e7}{change\+Active\+Profile}
\end{DoxyItemize}

You could pass an additional parameter "{}session\+\_\+write=true"{} to bypass this default. This read-\/only mode allow to use this API with parallel calls. In write mode, sessions are locked and your client must wait the end of a call before the next one can execute.
\item You can filter API access by enable the following parameters in \doxylink{class_g_l_p_i}{GLPI} General Configuration (API tab)\+:
\begin{DoxyItemize}
\item IPv4 range
\item IPv6 address
\item {\itshape App-\/\+Token} parameter\+: if not empty, you should pass this parameter in all of your API calls
\end{DoxyItemize}
\item \doxylink{class_session}{Session} and App tokens could be provided in query string instead of header parameters.
\end{DoxyItemize}\hypertarget{md_public_2glpi_2apirest_autotoc_md361}{}\doxysection{\texorpdfstring{Init session}{Init session}}\label{md_public_2glpi_2apirest_autotoc_md361}
