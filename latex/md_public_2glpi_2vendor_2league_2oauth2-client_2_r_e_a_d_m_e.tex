The OAuth 2.\+0 login flow, seen commonly around the web in the form of "{}\+Connect with Facebook/\+Google/etc."{} buttons, is a common integration added to web applications, but it can be tricky and tedious to do right. To help, we\textquotesingle{}ve created the {\ttfamily league/oauth2-\/client} package, which provides a base for integrating with various OAuth 2.\+0 providers, without overburdening your application with the concerns of \href{http://tools.ietf.org/html/rfc6749}{\texttt{ RFC 6749}}.

This OAuth 2.\+0 client library will work with any OAuth 2.\+0 provider that conforms to the OAuth 2.\+0 Authorization Framework. Out-\/of-\/the-\/box, we provide a {\ttfamily Generic\+Provider} class to connect to any service provider that uses \href{http://tools.ietf.org/html/rfc6750}{\texttt{ Bearer tokens}}. See our \href{https://oauth2-client.thephpleague.com/usage/}{\texttt{ basic usage guide}} for examples using {\ttfamily Generic\+Provider}.

Many service providers provide additional functionality above and beyond the OAuth 2.\+0 specification. For this reason, you may extend and wrap this library to support additional behavior. There are already many \href{https://oauth2-client.thephpleague.com/providers/league/}{\texttt{ official}} and \href{https://oauth2-client.thephpleague.com/providers/thirdparty/}{\texttt{ third-\/party}} provider clients available (e.\+g., Facebook, Git\+Hub, Google, Instagram, Linked\+In, etc.). If your provider isn\textquotesingle{}t in the list, feel free to add it.

This package is compliant with \href{https://github.com/php-fig/fig-standards/blob/master/accepted/PSR-1-basic-coding-standard.md}{\texttt{ PSR-\/1}}, \href{https://github.com/php-fig/fig-standards/blob/master/accepted/PSR-2-coding-style-guide.md}{\texttt{ PSR-\/2}}, \href{https://github.com/php-fig/fig-standards/blob/master/accepted/PSR-4-autoloader.md}{\texttt{ PSR-\/4}}, and \href{https://github.com/php-fig/fig-standards/blob/master/accepted/PSR-7-http-message.md}{\texttt{ PSR-\/7}}.
