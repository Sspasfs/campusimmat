{\footnotesize\ttfamily normalize (\begin{DoxyParamCaption}\item[{string}]{\$uri }\end{DoxyParamCaption})}

Takes a URI or partial URI as its argument, and normalizes it.

After normalizing a URI, you can safely compare it to other URIs. This function will for instance convert a \%7E into a tilde, according to rfc3986.

It will also change a \%3a into a \%3A.

{\footnotesize\ttfamily parse (\begin{DoxyParamCaption}\item[{string}]{\$uri }\end{DoxyParamCaption})}

Parses a URI and returns its individual components.

This method largely behaves the same as PHP\textquotesingle{}s parse\+\_\+url, except that it will return an array with all the array keys, including the ones that are not set by parse\+\_\+url, which makes it a bit easier to work with.

Unlike PHP\textquotesingle{}s parse\+\_\+url, it will also convert any non-\/ascii characters to percent-\/encoded strings. PHP\textquotesingle{}s parse\+\_\+url corrupts these characters on OS X.

\doxysubsubsection{\texorpdfstring{split()}{split()}}
{\footnotesize\ttfamily split (\begin{DoxyParamCaption}\item[{string}]{\$path }\end{DoxyParamCaption})}

Returns the \textquotesingle{}dirname\textquotesingle{} and \textquotesingle{}basename\textquotesingle{} for a path.

The reason there is a custom function for this purpose, is because basename() is locale aware (behaviour changes if C locale or a UTF-\/8 locale is used) and we need a method that just operates on UTF-\/8 characters.

In addition basename and dirname are platform aware, and will treat backslash () as a directory separator on Windows.

This method returns the 2 components as an array.

If there is no dirname, it will return an empty string. Any / appearing at the end of the string is stripped off.
