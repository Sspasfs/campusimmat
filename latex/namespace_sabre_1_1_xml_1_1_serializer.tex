This function is the \textquotesingle{}default\textquotesingle{} serializer that is able to serialize most things, and delegates to other serializers if needed.

The standard\+Serializer supports a wide-\/array of values.

\$value may be a string or integer, it will just write out the string as text. \$value may be an instance of Xml\+Serializable or Element, in which case it calls it\textquotesingle{}s xml\+Serialize() method. \$value may be a PHP callback/function/closure, in case we call the callback and give it the Writer as an argument. \$value may be a an object, and if it\textquotesingle{}s in the class\+Map we automatically call the correct serializer for it. \$value may be null, in which case we do nothing.

If \$value is an array, the array must look like this\+:

\mbox{[} \mbox{[} \textquotesingle{}name\textquotesingle{} =\texorpdfstring{$>$}{>} \textquotesingle{}\{namespace\+Uri\}element-\/name\textquotesingle{}, \textquotesingle{}value\textquotesingle{} =\texorpdfstring{$>$}{>} \textquotesingle{}...\textquotesingle{}, \textquotesingle{}attributes\textquotesingle{} =\texorpdfstring{$>$}{>} \mbox{[} \textquotesingle{}att\+Name\textquotesingle{} =\texorpdfstring{$>$}{>} \textquotesingle{}att\+Value\textquotesingle{} \mbox{]} \mbox{]} \mbox{[}, \textquotesingle{}name\textquotesingle{} =\texorpdfstring{$>$}{>} \textquotesingle{}\{namespace\+Uri\}element-\/name2\textquotesingle{}, \textquotesingle{}value\textquotesingle{} =\texorpdfstring{$>$}{>} \textquotesingle{}...\textquotesingle{}, \mbox{]} \mbox{]}

This would result in xml like\+:

\texorpdfstring{$<$}{<}element-\/name xmlns="{}namespace\+Uri"{} att\+Name="{}att\+Value"{}\texorpdfstring{$>$}{>} ... \texorpdfstring{$<$}{<}/element-\/name\texorpdfstring{$>$}{>} \texorpdfstring{$<$}{<}element-\/name2\texorpdfstring{$>$}{>} ... \texorpdfstring{$<$}{<}/element-\/name2\texorpdfstring{$>$}{>}

The value property may be any value standard\+Serializer supports, so you can nest data-\/structures this way. Both value and attributes are optional.

Alternatively, you can also specify the array using this syntax\+:

\mbox{[} \mbox{[} \textquotesingle{}\{namespace\+Uri\}element-\/name\textquotesingle{} =\texorpdfstring{$>$}{>} \textquotesingle{}...\textquotesingle{}, \textquotesingle{}\{namespace\+Uri\}element-\/name2\textquotesingle{} =\texorpdfstring{$>$}{>} \textquotesingle{}...\textquotesingle{}, \mbox{]} \mbox{]}

This is excellent for simple key-\/\texorpdfstring{$>$}{>}value structures, and here you can also specify anything for the value.

You can even mix the two array syntaxes.

The value\+Object serializer turns a simple PHP object into a classname.

Every public property will be encoded as an xml element with the same name, in the \doxylink{class_x_m_l}{XML} namespace as specified.

Values that are set to null or an empty array are not serialized. To serialize empty properties, you must specify them as an empty string.
